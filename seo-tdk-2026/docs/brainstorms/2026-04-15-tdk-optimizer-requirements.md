# TDK 自动优化工具 (Title/Description/Keywords Optimizer)

## Problem Frame

内容团队在编写页面 Title、Description、Keywords 时需要同时满足两个目标：
1. **编辑效率** — 快速生成符合 SEO 规范的 TDK，避免重复手工调整
2. **SEO 精度** — 确保 TDK 包含核心关键词、符合长度限制、无关键词堆砌，并与页面内容一致

当前的流程：编辑靠经验手工编写 → SEO 专家逐个审核 → 多轮反馈。这个流程重复、容易出错。

TDK 优化工具通过**规则驱动 + AI 辅助**的方式，在编辑端生成一个基于现有 SEO 最佳实践的 **最优 TDK 推荐**，同时通过自动检查（长度、堆砌、一致性）减少后续审核。目标是减少编辑的手工工作和 SEO 专家的审核负担。

## Requirements

**R1. 输入与基础信息**
- R1.1 支持输入：页面主题词、核心关键词（可多个）、页面内容摘要（可选）
- R1.2 支持手工修改上述信息，实时更新后续推荐

**R2. Title 生成与推荐**
- R2.1 基于用户输入（主题词、核心关键词、内容摘要），自动生成**一个最优 Title 推荐**
  - 遵循规范：核心词靠前、长度符合标准、避免堆砌
  - 用词自然、清晰表达页面价值
- R2.2 推荐同时生成 **2-3 个备选方案**（用户可选，可用于对比）
- R2.3 展示：Title 文本、长度计数（字符/汉字）、检查状态（✓ 通过 / ⚠ 警告）

**R3. Meta Description 生成与推荐**
- R3.1 基于用户输入，自动生成**一个最优 Description 推荐**
  - 遵循规范：包含核心关键词、长度符合标准、摘要清晰
  - 字数适中，能在搜索结果中完整显示
- R3.2 推荐同时生成 **2-3 个备选方案**（用户可选）
- R3.3 展示：Description 文本、长度计数、关键词覆盖状态

**R4. 长度检查与规则校验**
- R4.1 检查 Title 长度：
  - 英文标准：50-60 字符（用户可配置范围）
  - 中文标准：25-30 汉字（用户可配置范围）
  - 标记超长/过短警告
- R4.2 检查 Description 长度：
  - 英文标准：150-160 字符
  - 中文标准：75-80 汉字
  - 标记超长/过短警告
- R4.3 检查 Keywords 长度与个数：
  - 建议 5-8 个关键词，每个 5-20 字符
  - 标记过多/过少警告

**R5. 关键词堆砌检测**
- R5.1 检测 Title 中关键词重复出现，标记为"可能堆砌"（如同一词出现 2+ 次）
- R5.2 检测 Description 中关键词密度异常（超过合理范围），建议调整
- R5.3 提供修复建议（如移除重复词、分散关键词）

**R6. 标题与内容一致性检查**
- R6.1 输入页面内容摘要后，自动提取核心词（TF-IDF 或类似方法）
- R6.2 检查生成的 Title/Description 是否覆盖核心词
- R6.3 标记不一致项，建议调整

**R7. Keywords 生成与推荐**
- R7.1 基于主题词和核心关键词，自动生成推荐 Keywords 列表（8-12 个）
- R7.2 支持手工编辑和删除推荐词
- R7.3 根据推荐词生成逗号/英文分号分隔的 Keywords 字符串

## User Interfaces & Workflows

**用户角色与场景：**
1. **内容编辑** — 使用该工具快速生成 TDK，无需深度 SEO 知识
2. **SEO 专家** — 使用该工具批量审核、对比优化建议，并根据 SERP 竞争情况微调

**核心工作流：**
```
1. 编辑输入主题词、核心词、内容摘要
   ↓
2. 工具生成：
   - Title 最优推荐 + 2-3 个备选
   - Description 最优推荐 + 2-3 个备选
   - Keywords 推荐列表（8-12 个）
   ↓
3. 编辑查看推荐、验证检查结果（长度 ✓、堆砌 ✓、一致性 ✓）
   ↓
4. 编辑选择推荐或备选，手工微调（可选）
   ↓
5. 导出/发布最终 TDK
```

## Success Criteria
- **功能使用率** — 内容团队采用率 >30%（每周至少 2-3 次使用）
- **效率提升** — TDK 维护时间下降 20%（从平均 15 分钟/页面 → 12 分钟/页面）
- **质量保持或提升** — SEO 专家审核反馈次数 ≤ 当前水平，或检测到堆砌/不一致的问题数 ↓10%
- **易用性** — 内容编辑无需 SEO 培训可直接使用，操作流程 <2 分钟

## Scope Boundaries
- **非目标** — 不包括 SERP 实时排名抓取、竞品对标
- **非目标** — 不包括多语言自动翻译（但支持中英文规则分离）
- **集成范围** — 作为 Phase 6 系统的新模块，共享数据库和 API，生成的 TDK 直接保存到内容对象

## Key Decisions

| 决策 | 理由 |
|------|------|
| **单一最优推荐 + 2-3 备选** | 简化决策流程（编辑无需在多维度间权衡），同时保留灵活性；避免过度自动化 |
| **规则来源：从代码和文档提取** | 项目已有的 SEO 最佳实践，保证一致性，无需重新定义 |
| **集成 Phase 6** | 编辑在现有系统中使用，TDK 直接保存到内容对象，提升采用率；共享规则和数据库 |
| **生成按钮触发（非实时）** | 控制 API 成本和 UX 流畅度，避免每次输入变化都重新调用模型 |

## Dependencies / Assumptions
- 项目的 Phase 1-6 文档中有明确的 TDK 编写规范（需确认）
- 编辑和 SEO 专家都接受三维度（CTR/排名/平衡）的推荐方式
- AI 模型可访问（例如：Claude API，用于生成候选和一致性检查）
- 本地工具支持 Web UI 或 CLI
