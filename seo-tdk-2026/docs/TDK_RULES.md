# TDK 规范与验证规则

## 概述

本文档定义 TDK（Title、Description、Keywords）的编写规范和验证算法。这些规则用于：
1. **AI 生成阶段**：指导 Prompt，确保生成的候选符合规范
2. **验证阶段**：检查每个候选是否满足长度、堆砌、一致性等要求
3. **编辑参考**：帮助内容团队理解 SEO 最佳实践

---

## 1. Title（页面标题）

### 1.1 长度标准

| 语言 | 最优范围 | 最小 | 最大 | 说明 |
|------|---------|------|------|------|
| **英文** | 50-60 字符 | 30 | 70 | 搜索结果中显示 50-60 字符最佳 |
| **中文** | 25-30 汉字 | 15 | 40 | 搜索结果中显示 6-8 汉字（每汉字占 3-4 像素） |

**检查逻辑：**
- `length < minimum` → 失败 ✗（过短，不能充分表达主题）
- `minimum ≤ length ≤ optimal_max` → 通过 ✓（完美范围）
- `optimal_max < length ≤ maximum` → 警告 ⚠️（略长，可能被截断）
- `length > maximum` → 失败 ✗（过长，搜索结果中会被截断）

### 1.2 结构与关键词位置

**最佳实践：**
1. **核心词靠前**：主关键词应出现在标题前 1/3，帮助搜索引擎识别主题
2. **清晰表达**：避免堆积关键词，应用自然语言表达页面价值
3. **品牌词可选**：可在末尾添加品牌词（如"| MyBrand"）

**示例：**

✓ **好**
```
Python 教程：从入门到精通 | MyBrand
→ 核心词"Python 教程"靠前，价值清晰
```

✗ **不好**
```
Python、教程、学习 Python、在线 Python 教程 - MyBrand
→ 堆砌，读起来不自然
```

### 1.3 避免堆砌

**定义**：关键词在标题中重复出现或密度过高

**检测规则**：
- 同一词重复 2 次或以上 → 警告 ⚠️
- 同一词重复 3 次或以上 → 失败 ✗
- 关键词密度 > 10% → 警告 ⚠️
- 关键词密度 > 15% → 失败 ✗

**关键词密度公式**：
```
density = (keyword_count × keyword_length) / title_length × 100%
```

**示例：**

✓ **通过**
```
标题：Python 教程：零基础学 Python 编程
- "Python" 出现 2 次，但自然，不算堆砌
- 密度 = (2 × 6) / 25 ≈ 48%... 等等，这个例子有问题，让我重新计算

实际上应该计算：关键词占比 (keyword_count) / word_count
标题："Python 教程：零基础学 Python 编程"（7 个词）
- "Python" 出现 2 次
- 密度 = 2 / 7 ≈ 28% 仍然高，但这是合理的多词标题

让我重新定义更清晰的算法...
```

**重新定义堆砌检测算法**：

1. **重复词检测**（优先级高）：
   - 同一个词重复 ≥ 3 次 → 失败 ✗
   - 同一个词重复 = 2 次 → 检查是否自然（如果是主关键词，允许）

2. **单词密度检测**（针对单个字符序列）：
   - 计算每个关键词在 title 中的出现频率
   - 频率 > 20% → 可能堆砌 ⚠️

**示例（改进）：**

✓ **通过**
```
标题（中文）："Python 编程教程 - 从零开始学 Python"
- "Python" 出现 2 次（主关键词，允许）
- "教程" 出现 1 次
- 检测结果：无堆砌 ✓
```

✗ **失败**
```
标题（中文）："Python 学习 Python 教程 Python 编程"
- "Python" 出现 3 次 → 失败 ✗（过度重复）
```

## 2. Meta Description（元描述）

### 2.1 长度标准

| 语言 | 最优范围 | 最小 | 最大 | 说明 |
|------|---------|------|------|------|
| **英文** | 150-160 字符 | 100 | 200 | 搜索结果摘要显示 120-160 字符 |
| **中文** | 75-80 汉字 | 50 | 100 | 搜索结果摘要显示 20-26 汉字 |

**检查逻辑**：同 Title，但数值不同

### 2.2 结构与关键词覆盖

**最佳实践：**
1. **包含核心关键词**：前 30 字内应包含主关键词
2. **清晰表达价值**：描述页面的主要内容和用户收益
3. **自然可读**：避免列表堆砌，应读起来像完整的句子
4. **包含 CTA（可选）**：可末尾添加行动号召（如"立即阅读"、"了解更多"）

**示例：**

✓ **好**
```
学习 Python 编程的完整教程。从基础语法、数据结构、到面向对象编程，
逐步掌握 Python 核心技能。适合初学者和进阶开发者。
→ 清晰表达页面内容，包含核心关键词（Python、教程、编程）
```

✗ **不好**
```
Python、编程、教程、学习 Python、在线 Python、Python 代码、...
→ 堆砌，无法表达实际内容价值
```

### 2.3 关键词覆盖检查

**定义**：Description 中是否包含标题中的核心关键词

**检测规则**：
- 标题中的每个关键词（停用词除外）应至少在 Description 中出现 1 次
- 若核心关键词（用户指定的）缺失 → 警告 ⚠️

**示例：**

✓ **覆盖完整**
```
Title: "Python 编程教程"
Description: "学习 Python 编程的完整教程..."
→ 关键词 Python ✓、编程 ✓、教程 ✓ 都出现
```

⚠️ **覆盖不足**
```
Title: "Python 编程教程"
Description: "学习如何编程的完整课程..."
→ 缺少核心关键词 Python ⚠️
```

## 3. Keywords（关键词标签）

### 3.1 数量与长度

| 项目 | 推荐值 | 说明 |
|------|--------|------|
| **关键词数量** | 5-8 个 | 过多无益，现代搜索引擎权重已降低 |
| **单个关键词长度** | 5-20 字符（中文 2-10 汉字） | 避免过短或过长的短语 |

### 3.2 关键词来源与排序

**来源优先级：**
1. **用户指定关键词**（权重最高）
2. **标题中提取的词** （权重次高）
3. **Description 中提取的词** （权重次）
4. **页面内容主题词** （权重最低）

**排序规则**：
1. 按相关性排序（用户指定 > 自动提取）
2. 按频率排序（在页面中出现次数多的词靠前）

**示例：**

✓ **合理**
```
用户指定：["Python", "编程", "教程"]
推荐补充：["学习", "初学者", "代码示例"]
最终：["Python", "编程", "教程", "学习", "初学者", "代码示例"]
```

### 3.3 避免关键词堆砌（Keywords 字段）

**检测规则**：
- 关键词重复 → 警告 ⚠️（Keywords 字段中不应重复）
- 单个关键词过长（>20 字符）→ 警告 ⚠️

## 4. 一致性检查（Title/Description ↔ 页面内容）

### 4.1 核心词提取（MVP 实现）

**算法**：简单词频统计 + 停用词过滤

**步骤**：
1. 分词：将页面内容按词（中文按字或短语）分割
2. 过滤停用词：移除"的"、"和"、"是"、"在"等无意义词
3. 计数：统计每个词的出现频率
4. 排序：按频率排序，取 Top 5-8 个词作为核心词

**停用词列表（示例）：**
```
中文：的、和、是、在、有、被、对、与、还是、也、或、及、但、等、所、这、那、了、着、过、要、会、可、能、...
英文：the、a、an、and、or、but、in、on、at、is、are、was、were、be、do、does、did、...
```

### 4.2 一致性检查规则

**定义**：Title 和 Description 是否覆盖了页面的核心主题词

**检测规则**：
1. 提取页面内容的 Top 5 核心词
2. 检查 Title + Description 是否至少包含 3 个核心词
3. 覆盖度计算：`(matched_words / top_words) × 100%`

**判定标准**：
- 覆盖度 ≥ 80% → 通过 ✓
- 覆盖度 60-80% → 警告 ⚠️（部分不一致）
- 覆盖度 < 60% → 信息 ℹ️（大幅不一致，需注意）

**示例：**

✓ **高一致性**
```
页面内容摘要："这是一篇关于 Python 编程的教程。文章涵盖语法、...、数据结构、...、对象编程。"
核心词（Top 5）：["Python", "编程", "教程", "语法", "数据结构"]
Title："Python 编程教程"
Description："学习 Python 编程...数据结构..."
覆盖词：["Python" ✓, "编程" ✓, "教程" ✓, "语法" ✓, "数据结构" ✓]
覆盖度：100% ✓
```

⚠️ **低一致性**
```
页面内容核心词：["Python", "编程", "教程", "语法", "数据结构"]
Title："网络编程入门"
Description："学习网络通信..."
覆盖词：["编程" ✓] 仅 1 个
覆盖度：20% ⚠️（严重不一致）
```

## 5. 规则参数与阈值速查

### 5.1 可配置阈值

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `TITLE_LENGTH_MIN_EN` | 30 | 英文 Title 最小字符 |
| `TITLE_LENGTH_OPTIMAL_EN` | 50-60 | 英文 Title 最优范围 |
| `TITLE_LENGTH_MAX_EN` | 70 | 英文 Title 最大字符 |
| `TITLE_LENGTH_MIN_ZH` | 15 | 中文 Title 最小汉字 |
| `TITLE_LENGTH_OPTIMAL_ZH` | 25-30 | 中文 Title 最优范围 |
| `TITLE_LENGTH_MAX_ZH` | 40 | 中文 Title 最大汉字 |
| `DESC_LENGTH_MIN_EN` | 100 | Description 最小字符 |
| `DESC_LENGTH_OPTIMAL_EN` | 150-160 | Description 最优范围 |
| `DESC_LENGTH_MAX_EN` | 200 | Description 最大字符 |
| `DESC_LENGTH_MIN_ZH` | 50 | Description 最小汉字 |
| `DESC_LENGTH_OPTIMAL_ZH` | 75-80 | Description 最优范围 |
| `DESC_LENGTH_MAX_ZH` | 100 | Description 最大汉字 |
| `STACKING_REPEAT_THRESHOLD` | 3 | 同词重复 N 次判为堆砌 |
| `STACKING_DENSITY_WARN` | 15% | 词密度 >15% 警告 |
| `STACKING_DENSITY_FAIL` | 25% | 词密度 >25% 失败 |
| `CONSISTENCY_COVERAGE_PASS` | 80% | 一致性覆盖度 ≥80% 通过 |
| `CONSISTENCY_COVERAGE_WARN` | 60% | 一致性覆盖度 ≥60% 警告 |

### 5.2 验证结果状态

| 状态 | 图标 | 颜色 | 含义 |
|------|------|------|------|
| `pass` | ✓ | 绿色 | 符合规范 |
| `warn` | ⚠ | 黄色 | 警告，可能需要优化 |
| `fail` | ✗ | 红色 | 不符合规范，建议修改 |
| `info` | ℹ | 蓝色 | 信息提示（仅一致性检查使用） |

## 6. 实现示例

### 6.1 Title 验证示例

**输入：**
```typescript
const title = "Python 教程：从零开始学 Python";
const language = "zh";
```

**执行：**
```
1. 计算长度：12 汉字
2. 长度检查：12 < 15（最小值）→ 失败 ✗

输出：
{
  status: 'fail',
  message: '标题过短（12 汉字），建议 15-40 汉字',
  length: 12,
  optimal_range: '25-30',
  min: 15,
  max: 40
}
```

### 6.2 堆砌检测示例

**输入：**
```typescript
const title = "Python 学习 Python 教程 Python 编程";
const keywords = ["Python", "学习", "教程", "编程"];
```

**执行：**
```
1. 分词：["Python", "学习", "Python", "教程", "Python", "编程"]
2. 计数：
   - "Python": 3 次 → 失败（≥3）
   - "学习": 1 次 → 通过
   - "教程": 1 次 → 通过
   - "编程": 1 次 → 通过
3. 密度：
   - "Python": 3 / 6 = 50% > 25% → 失败

输出：
{
  status: 'fail',
  issues: [
    '"Python" 出现 3 次，过度重复',
    '关键词密度 50% 过高（>25%）'
  ]
}
```

### 6.3 一致性检查示例

**输入：**
```typescript
const contentSnippet = "本文介绍 Python 编程基础。涵盖变量、数据类型、控制流、函数等核心概念。适合初学者学习 Python。";
const title = "Python 编程教程";
const description = "学习 Python 基础编程知识...";
```

**执行：**
```
1. 提取核心词（Top 5）：
   ["Python", "编程", "基础", "数据", "函数"]
   
2. 检查 Title + Description：
   - "Python" 在 Title ✓ 和 Description ✓ 中出现
   - "编程" 在 Title ✓ 中出现
   - "基础" 在 Description ✓ 中出现
   - "数据" 未出现 ✗
   - "函数" 未出现 ✗

3. 覆盖度：3 / 5 = 60% → 警告

输出：
{
  status: 'warn',
  coverage: 60%,
  matched_words: ["Python", "编程", "基础"],
  missing_words: ["数据", "函数"],
  message: '一致性 60%。建议补充关键词："数据"、"函数"'
}
```
