# 重组 OpenClaw 周报——从业务成果视角的宏观呈现

## 问题框架

**现状**：5 份部门周报独立存在，各含 500-1500 字。数据丰富但分散，缺乏宏观视角和说服力故事线。

**目标**：融合为**1 份** 3000-4000 字的宏观周报，用业务成果为开篇（破题），逐层展开细节，让领导一眼看到"这周团队干了什么有形的事"。

**成功标准**：
- ✅ 开篇 1 分钟能说出核心成果（从数据和项目两个维度）
- ✅ 保留所有关键细节（部门数据、风险警示、下周方向），但嵌入在故事中而非平铺
- ✅ 跨部门数据能统一呈现（总体 50+ 智能体 / 750+ 触发等），同时保留部门特色
- ✅ 可视化和转折点清晰，让阅读保持节奏

## 实现细节

### 信息优化规则

**提取**：从 5 份原始周报中提取每个部分的最强 3-5 个数据点和故事点

**整合**：
- 相同主题的数据（如"节省人工"）做求和与平均化
- 部门特色保留（豆腐的体系性、Jimmy 的项目性、嘎嘎的数据驱动性）
- 风险信息集中到一个转折点，而不是散在各处

**可视化设计**：
- Section 2 的指标面板用表格（便于对标）
- Section 4 的风险用时间轴或倒计时视觉（制造紧迫感）
- Section 3 的部门数据可用简单的对比图（智能体数、触发数、节省时数）

### 转折点与节奏控制

为了让整份周报"引人入胜"，在以下位置加入心理转折：

1. **成果高光** → **数据支撑**：从"我们做了什么"到"规模有多大"
2. **系统规模** → **部门落实**：从"宏观数据"到"具体是谁、怎么做的"
3. **部门细节** → **风险转折**：从"成就感"到"我们还需要注意什么"
4. **风险警示** → **下周希望**：从"挑战"到"明确的解决方向"
